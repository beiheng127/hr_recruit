# 人力资源招聘管理系统 - 故障记录

> 记录开发过程中多次出现的故障模式、根因分析与修复方案，避免重复踩坑。

## 故障 001：Controller 路径缺少 `/api` 前缀

**现象：** 前端请求 `POST /api/auth/login` 返回 `404 No static resource api/auth/login`

**根因：**
- 前端 `request.js` 的 `baseURL: '/api'`，所以所有 API 调用自动加 `/api` 前缀
- 但后端 `AuthController` 的 `@RequestMapping("/auth")` 只处理 `/auth/**`
- `SecurityConfig` 放行的是 `/api/auth/**`，请求过了 Security 但被当成静态资源处理

**修复：** 所有 Controller 的 `@RequestMapping` 统一加 `/api` 前缀
- `AuthController`: `/auth` → `/api/auth`
- `JobPositionController`: `/jobs` → `/api/jobs`
- 新建 Controller 也必须遵循此约定

**涉及文件：**
- `AuthController.java`
- `JobPositionController.java`
- `ResumeController.java`
- `AiController.java`
- `DashboardController.java`

## 故障 002：前端字段名与后端实体类字段名不一致

**现象：** 表格有列头但无数据显示，或显示为空白/undefined

**根因：** 前端代码中 `prop="xxx"` 或表单字段名与后端实体类字段名不匹配

**具体案例：**

### 岗位管理 (JobListView.vue)
| 前端旧字段 | 后端实体字段 | 修复 |
|-----------|------------|------|
| `name` | `positionName` | ✓ 已修复 |
| `responsibilities` | `responsibility` | ✓ 已修复 |
| `requirements` | `requirement` | ✓ 已修复 |
| `salaryMin/salaryMax` | `salaryRange` | ✓ 已修复 |

### 简历管理 (ResumeListView.vue)
| 前端旧字段 | 后端实体字段 | 修复 |
|-----------|------------|------|

2. 字符串技能标签需用 `split(/[,，、]/)` 转为数组渲染 Tag
3. 编辑表单字段也要同步更新

## 故障 003：分页数据解析字段错误

**现象：** 表格数据为空，分页显示 Total 0

**根因：** MyBatis-Plus 的 `Page<T>` 对象返回的数据在 `records` 字段，而前端按 `list` 解析

**修复：**
```js
// 错误
res.data?.list || res.data || []

// 正确
res.data?.records || res.data?.list || res.data || []
```

**涉及文件：**
- `JobListView.vue`
- `ResumeListView.vue`

---

## 故障 004：分页请求参数名不匹配

**现象：** 后端接收不到分页参数，始终返回第一页

**根因：** 后端 `Page<T>` 默认使用 `pageNum` / `pageSize`，前端传了 `page` / `pageSize`

**修复：**
```js
// 错误
{ page: pagination.page, pageSize: pagination.pageSize }

// 正确
{ pageNum: pagination.page, pageSize: pagination.pageSize }
```

**涉及文件：**
- `JobListView.vue`
- `ResumeListView.vue`

## 故障 005：CSS 全局样式污染其他页面

**现象：** 岗位管理/简历管理页面加载后，登录页、侧边栏等样式被覆盖

**根因：** `global.css` 中对 Element Plus 组件使用了全局选择器（如 `.el-table`、`.el-tag`），没有作用域限定

**修复：** 将全局样式改为只在主内容区生效：
```css
/* 错误：全局污染 */
.el-table { border-radius: 12px !important; }

/* 正确：限定作用域 */
.main-content .el-table { border-radius: 12px !important; }
```

**涉及文件：**
- `src/styles/global.css`

## 故障 006：Result.success(String) 类型不匹配

**现象：** 编译错误 `incompatible types: Result<Void> cannot be converted to Result<String>`

**根因：** `Result.success("xxx")` 调用的是返回 `Result<Void>` 的重载方法，但 Controller 方法声明返回 `Result<String>`

**修复：** 使用带 data 参数的重载：
```java
// 错误
return Result.success("暂无面试记录");  // 返回 Result<Void>

// 正确
return Result.success("暂无面试记录", "暂无面试记录");  // 返回 Result<String>
```

**涉及文件：**
- `InterviewController.java`

## 故障 007：Spring AI ChatClient Bean 不存在

**现象：** 启动报错 `No qualifying bean of type 'ChatClient' available`

**根因：** Spring AI 1.0.0-M4 只自动创建 `ChatModel`，不自动创建 `ChatClient`

**修复：** 手动创建 Bean
```java
@Bean
ChatClient chatClient(ChatModel chatModel) {
    return ChatClient.create(chatModel);
}
```

**涉及文件：**
- `config/AiConfig.java` (新建)

## 故障 008：数据库编码配置错误

**现象：** 启动报错 `Unknown character set index for field '255' received from server`

**根因：** JDBC URL 中 `characterEncoding=utf8mb4` 是 MySQL 字符集名，但 JDBC 驱动只识别 `UTF-8`

**修复：**
```yaml
# 错误
url: jdbc:mysql://...?characterEncoding=utf8mb4

# 正确
url: jdbc:mysql://...?characterEncoding=UTF-8
```

**涉及文件：**
- `application.yml`

## 故障 009：Flyway 迁移但数据库不存在

**现象：** 启动报错 `Unknown database 'hr_recruit'`

**根因：** Spring Boot 应用启动时会尝试连接数据库，但数据库尚未创建

**修复：** 手动创建数据库
```sql
CREATE DATABASE hr_recruit CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

**涉及文件：**
- 无，需手动执行 SQL

## 故障 010：前端 import 命名冲突

**现象：** 编译报错 `Identifier 'generateJd' has already been declared`

**根因：** 从 API 导入的函数名与组件内部方法名相同

**修复：** 使用别名导入
```js
// 错误
import { generateJd } from '@/api/ai'

// 正确
import { generateJd as generateJdApi } from '@/api/ai'
```

**涉及文件：**
- `AiCenterView.vue`

## 故障 011：SSE 流式响应格式问题

**现象：** AI 对话流式输出时，前端收不到数据或数据格式错误

**根因：**
1. 后端使用 `Flux<String>` 返回 SSE，但前端用普通 `axios.post` 请求
2. Spring AI 的 SSE 格式是 `data: content\n\n`，前端需要 `EventSource` 解析

**修复：**
- 后端保持 `produces = "text/event-stream"`，返回 `Flux<String>`
- 前端改用 `new EventSource('/api/ai/chat?message=xxx')` 接收 SSE
- 注意处理 `[DONE]` 或空数据作为结束标志

**涉及文件：**
- `AiController.java`
- `AiCenterView.vue`

## 故障 013：SecurityConfig 路径不匹配 + CORS

**现象：** 前端报 CORS 错误或 401

**根因：**
1. `SecurityConfig` 中放行路径写错：`/auth/**` 应为 `/api/auth/**`
2. CORS `allowedOrigins("*")` 与 `allowCredentials(true)` 冲突

**修复：**
```java
// 路径
.requestMatchers("/api/auth/**", "/swagger-ui/**").permitAll()

// CORS
config.setAllowedOrigins(List.of("http://localhost:5173", "http://127.0.0.1:5173"));
```

**涉及文件：**
- `SecurityConfig.java`

## 预防 checklist

每次新增页面/功能时检查：
- [ ] 后端 Controller 的 `@RequestMapping` 是否带 `/api` 前缀
- [ ] 前端表格 `prop` 是否与后端实体字段名完全一致
- [ ] 分页参数是否用 `pageNum` / `pageSize`
- [ ] 分页数据是否从 `res.data.records` 解析
- [ ] CSS 是否加了作用域限定（`.main-content` 或 `scoped`）
- [ ] API 导入函数是否有命名冲突
- [ ] 后端返回 `Result<T>` 时类型是否匹配
- [ ] 前后端状态值是否统一（英文枚举 vs 中文标签）

## 故障 014：前后端状态值不匹配

**现象：** 状态筛选失效、状态标签显示错误、操作按钮判断错误

**根因：** 后端使用英文状态枚举（如 `DRAFT`/`PUBLISHED`/`CLOSED`），但前端使用中文状态值（如 `草稿`/`已发布`/`已关闭`）进行筛选、显示和逻辑判断

**具体案例：**

### 岗位管理 (JobListView.vue)
| 位置 | 前端值 | 后端值 | 修复 |
|-------|---------|--------|------|
| 状态筛选下拉框 | `value="草稿"` | `value="DRAFT"` | ✓ 已修复 |
| 状态标签显示 | `{{ row.status }}` | 需显示中文 | ✓ 已修复（新增`statusLabel()`） |
| 操作按钮判断 | `row.status === '已发布'` | `row.status === 'PUBLISHED'` | ✓ 已修复 |

### 简历管理 (ResumeListView.vue)
| 位置 | 前端值 | 后端值 | 修复 |
|-------|---------|--------|------|
| 人才库状态筛选 | `value="人才库"` | `value="TALENT"` | ✓ 已修复 |
| 移动到下拉菜单 | `command="人才库"` | `command="TALENT"` | ✓ 已修复 |
| 状态标签显示 | `{{ row.talentPoolStatus }}` | 需显示中文 | ✓ 已修复（新增`talentStatusLabel()`） |

**后端状态枚举值：**

| 实体 | 字段 | 英文值 | 中文标签 |
|-------|--------|---------|----------|
| JobPosition | status | DRAFT | 草稿 |
| JobPosition | status | PUBLISHED | 已发布 |
| JobPosition | status | CLOSED | 已关闭 |
| ResumeInfo | talentPoolStatus | PENDING | 待处理 |
| ResumeInfo | talentPoolStatus | NORMAL | 正常 |
| ResumeInfo | talentPoolStatus | TALENT | 人才库 |
| ResumeInfo | talentPoolStatus | INTERVIEW | 面试中 |
| ResumeInfo | talentPoolStatus | HIRED | 已录用 |
| ResumeInfo | talentPoolStatus | BLACKLIST | 黑名单 |
| InterviewArrangement | status | SCHEDULED | 待面试 |
| InterviewArrangement | status | COMPLETED | 已完成 |

**修复原则：**
1. 前端筛选/提交使用英文值（与后端一致）
2. 前端显示使用中文标签（通过映射函数转换）
3. 前端逻辑判断使用英文值

**涉及文件：**

- `src/views/resume/ResumeListView.vue` ✓ 已修复

## 故障 015：CandidateController 类型转换错误

**现象：** 编译错误，类型不匹配

**根因：** `ResumeInfoService.uploadAndParse()` 返回 `Long`（resumeInfoId），但 `CandidateController.apply()` 中错误地将其返回值当作 `String` 使用，且语义不正确（投递简历不应触发完整的简历解析流程）

**修复：**
1. 新增 `ResumeInfoService.uploadFile()` 方法：仅上传文件，返回 `resumeFileId`
2. `CandidateController.apply()` 改为调用 `uploadFile()`
3. 更新 `ResumeInfoService` 接口定义

**涉及文件：**
- `ResumeInfoService.java`
- `ResumeInfoServiceImpl.java`
- `CandidateController.java`

## 故障 017：前端候选人页面 API 路径错误

**现象：** 候选人门户页面 API 请求 404

**根因：** 候选人页面使用了需要 JWT 认证的 `/api/jobs/**` 路径，但实际应该使用公开的 `/candidate/jobs/**` 路径

**修复：**
- `CandidateApplyView.vue`: `/api/jobs/${jobId}` → `/candidate/jobs/${jobId}`
- `CandidateApplyView.vue`: `/api/candidate/apply` → `/candidate/apply`
- `CandidateJobListView.vue`: 改用 `request.get('/candidate/jobs', ...)`
- `CandidateJobDetailView.vue`: `/api/jobs/${id}` → `/candidate/jobs/${id}`

**涉及文件：**
- `src/views/candidate/CandidateApplyView.vue` ✓ 已修复
- `src/views/candidate/CandidateJobListView.vue` ✓ 已修复
- `src/views/candidate/CandidateJobDetailView.vue` ✓ 已修复
