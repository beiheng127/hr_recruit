## 回归测试要点

### 状态流转一致性
```
applicant_job.status 流转：
  SCREENING → INTERVIEW → OFFER → HIRED
  SCREENING → INTERVIEW → REJECTED

面试创建时自动创建：
  applicant_job（投递记录）
  candidate_stage_record（阶段记录）
  interview_arrangement（面试安排）

Offer 发送时同步：
  offer_record.status = SENT
  applicant_job.status = OFFER

Offer 接受时同步：
  offer_record.status = ACCEPTED
  applicant_job.status = HIRED
```

### 数据库检查项
- [ ] ai_match_record 表有匹配记录数据
- [ ] interview_evaluation.recommend 字段有值
- [ ] notification 表数据按 recipient_id 隔离正确
- [ ] recruitment_pipeline 表有 deleted 列
- [ ] offer_record.salary 字段支持字符串格式（如 "30k*14薪"）
